05-1. 폼 삽입하기

웹에서 form(폼)은 아이디, 비밀번호등을 입력해 로그인 버튼, 회원 가입 등 사용자가 웹사이트로 정보를 보낼수 있는 요소 전부.

1. 로그인 폼의 동작 과정

사용자(id,비밀번호 입력)
-> 로그인 버튼 클릭(입력한 정보 웹 서버로 전송)
-> 웹 서버(가진 데이터베이스에서 입력받은 id,비빌번호 일치하는지 확인 결과 전송)
-> 웹 브라우저(결과 노출)

폼과 관련한 작업은 정보 저장, 검색, 수정이 대부분이라 데이터베이스를 기반으로 작동.
화면에 노출되는 폼 형태는 HTML태그, CSS로 만들지만 폼에 입력한정보는 ASP,PHP,JSP같은 서버 프로그래밍을 이용해 처리.

2. 폼을 만드는 <form> 태그

기본형 : <form [속성="속성값"]>여러 폼 요소</form>
<form>태그의 속성
속성 설명
method 사용자가 입력한 내용을 서버 프로그램으로 어떤 방식으로 넘겨줄것인지 설정.

속성 값: get, post
- get : 입력 길이 제한있음.주소표시줄에 사용자가 입력한 내용이 노출되는 단점이 있음.
- post : 입력 길이 제한이 없고, 사용자 입력한 내용도 드러나지 않음.
name 자바스크립트로 폼을 제어할 때 사용할 이름 지정.
actiion 폼 태그 안의 내용을 처리해 줄 서버 프로그램 지정.
target action 속성에서 지정한 스크립트 파일을 현재 창이 아닌 다른 위치에서 엶.
autocomplete 사용자가 입력한 내용을 기억했다가 비슷한 글자가 입력되면 이전에 입력한 내용을 드롭다운으로 보여줌.

기본 값 "on"
중요한 개인정보를 사용할땐 속성값을 "off"로 지정해 자동완성기능을 끈다.

e.g. 폼에 내용을 입력하고 서버로 전송했을때 서버에 있는 regiser.php를 실행
<form action="register.php">
  여러 폼 요소 (input)
</form>

4. 폼 요소에 레이블을 붙이는 <label></label> 태그의 2가지 사용법

(1). <label>태그는 <input>입력란 가까이에 input값이 뭔지 제목을 붙여놓은 텍스트. 폼요소와 레이블 텍스트가 연결되었다는걸 웹브라우저가 알수있음.

기본형 : <label>레이블명<input></label>

(2). <label> 태그와 <input> 를 따로 사용하여 각 for 과 id 속성의 값을 동일하게 설정해 연결. 서로 떨러져있어도 쉽게 연결 가능.label에 css 를 설정해 input을 꾸밈.

기본형 :
<label for="id명">레이블명</label>
<input id="id명">

05-2. 사용자의 입력을 위한 input태그

아이디, 검색상자, 로그인 버튼등 사용자가 입력학 부분은 <input> 태그를 이용해 넣음.
<input> 태그는 다양한 폼에서 사용자가 입력한 정보를 받을 때 사용.

input 태그의 type 속성

(1). type = "text", "password"

- size : 화면에 몇 글자가 보이도록 할 것인지 지정.
   최대 입력 글자수가 10개, size 5개
   -> 필드크기 5개 글자에 맞추고 나머지는 화면에 안보임.
- minlength, maxlength : 필드에 입력할 수 있는 최소, 최대 문자수 지정.
- value : 필드에 표시할 내용. 사용 안하면 빈 텍스트 필드. password 필드에서 사용안함.


(2). type = "search", "url", "email", "tel"


다양한 용도에 맞게 입력할 수 있는 텍스트필드(HTML5부터)
PC에선 큰변화가 없으나 모바일 웹 브라우저에서는 이메일, 전화번호에따라 가상키보드 배열이 바뀜.


(3). type = "checkbox", "radio" checkbox : 주어진 여러 항목에서 2개이상 선택가능 /
radio : 묶을 radio의 name 값을 똑같이 지성하면 주어진 여러 항목에서 1개만 선택가능 /

- value : 선택한 체크박스, 라디오버튼을 서버에게 알려 줄 때 넘겨줄 값 지정.
   영문, 숫자여야 하며 필수 속성
- checked : 기본으로 선택해 놓고 싶은 항목이 있으면 사용.

- step : 숫자 간격을 지정, 기본값 1.
- value : 필드에 표시할 초깃값.

(6). type = "time", "datetime-local"

- min : 날짜나 시간의 범위 제한, 범위의 시작 날짜나 시간.
- max : 날짜나 시간의 범위 제한, 범위의 마지막 날짜나 시간.
- step : 스핀 박스의 화살표를 클릭했을 때 증감시킬 크기.

05-3. input 태그의 주요 속성

1. autofocus

페이지를 불러오자마자 폼에서 원하는 요소에 마우스 포인터를 표시

2. placeholder

입력란에 힌트 내용 표시, 필드를 클릭하면 사라짐

3. readonly

읽기 전용 필드, readonly or readonly="readonly" or readonly="true"

4. required

필수입력 입력을 안하면 submit 전송이 안됨

05-4. 폼에서 사용하는 여러 가지 태그

1. <textarea></textarea>

텍스트 영역, 폼에서 텍스트를 여러줄 입력하는 영역. 게시판에서 글을 입력하거나 회원 가입 양식에서 사용자 약관을 표시할때 자주 사용.
- cols : 텍스트 영역 가로 너비를 문자 단위로 지정.
- row : 텍스트 영역의 세로 길리를 줄 단위로 지정, 지정한 숫자보다 줄 개수가 많아시면 스크롤 막대가 생김.

2. <select></select>, <option>

여러 옵션중에 선택할 수 있게 드롭다운 데이터 목록을 보여줌.

1. <select> :
size : 드롭다운 항목의 개수 지정
multiple : 드롭다운 목록에서 둘 이상의 항목 선택
2. <option> : 드롭다운 목록에 표시되는 옵션
value : 해당 항목을 선택할 때 서버로 넘겨줄 값 지정
selected : 기본적으로 선택해서 보여줄 항목

3. <datalist></datalist>, <option>

값을 직접 입력하지 않고 미리 만들어 놓은 값 중에서 선택가능. option의 value로 선택되고 부가 설명이 보임

4. <button></button>

<button> 태그를 이용하여 폼을 전송하거나 리셋하는 버튼 삽입.
화면 낭독기로 웹 문서를 읽어 줄 때 <button> 태그를 만나면 이 부분에 버튼이 있다는 것을 정확히 전달함.
버튼이 필요한 어디든지 활용, <button> 태그에는 콘텐츠를 포함할 수 있어서 아이콘추가나 CSS로 원하는형태 꾸미기 가능.

- type="submit" : 폼을 서버로 전송, <input type="submit"> 과 같은 기능.
- type="reset" : 폼에 입력한 내용을 초기화, <input type="reset"> 과 같은 기능.
- type="button" : 버튼 형태만 만들뿐 자체기능은 없음, <input type="submit"> 과 같은 기능.

05-5. web.dev 유효성 검사 규칙 정의 | ( ✈ wed.dev page )

1.required 속성 : input 태그의 required 속성은 폼 데이터가 서버로 제출되기 전 반드시 채워져 있어야 하는 입력 필드 명시.

<lavel for="password">minlength로 최소 작성 글자 수 정의<label>
<input required minlength="8" type="password" id="password" name="password">

<lavel for="animal">정규식 패턴 으로 규칙 정의<label>
<input required pattern="[a-z]{2,20}" type="text" id="animal" name="animal">

2. required 속성 선택자

<lavel for="name">required 속성 선택자<label>
<input required type="text" id="name" name="name">

input#name:invalid {border: 2px solid red;}
